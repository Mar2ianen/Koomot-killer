# Product Brief

## Рабочее название

**Komoot Killer**

## Проблема

Существующие вело-картографические сервисы часто ощущаются как медленные, закрытые и завязанные на подписки. Для продвинутого пользователя маршрут должен быть не просто линией на карте, а понятным инженерным объектом: где покрытие, где набор, где опасные дороги, где сомнительные участки, где OSM-данные неполные.

## Цель MVP

Сделать быстрый локальный GPX viewer / route inspector:

- открыть `.gpx`;
- показать трек на карте;
- посчитать дистанцию, набор, сброс и высотный профиль;
- разбить маршрут на сегменты;
- показать предупреждения;
- подготовить основу для будущего OSM surface analyzer и роутинга.

## Позиционирование

Не аналог Komoot «один в один», а инструмент для тех, кто хочет понимать маршрут.

Формула:

> VLC для маршрутов: открыл GPX, увидел правду, поправил, экспортнул, поехал.

## Пользовательские сценарии

### 1. Проверить GPX перед поездкой

Пользователь скачивает GPX с гонки или из внешнего сервиса, открывает его, смотрит дистанцию, профиль высоты, сложные места и потенциальные проблемы.

### 2. Быстро оценить маршрут для гревела

Пользователь хочет понять, насколько маршрут подходит под 40–45 мм резину, где могут быть плохие участки и где будут тяжёлые сегменты.

### 3. Собрать базу маршрутов

Пользователь хранит локальные маршруты, открывает их без привязки к подписке и экспортирует обратно в GPX.

### 4. Подготовить маршрут к будущему роутингу

Сначала приложение работает с готовыми GPX. Позже появляется OSM-анализ, редактор и собственные профили маршрутизации.

## Killer value

- скорость;
- локальность;
- GPX-first подход;
- понятная аналитика;
- открытая архитектура;
- отсутствие подписочной зависимости для базовых функций.
